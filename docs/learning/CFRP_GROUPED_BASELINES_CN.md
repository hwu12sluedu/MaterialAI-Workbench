# CFRP 分组基线：从真实数据到可信代理模型

本节接续 [CFRP 公开实验数据：受控导入与可信划分](CFRP_EXPERIMENTAL_DATASET_CN.md)。目标不是调出一个漂亮分数，
而是完整走通“固定数据版本、固定验证折、多个基线、逐样本证据、误差解释、模型落盘”的机器学习闭环。

## 1. 本节学完能回答什么

1. 为什么同一种 CFRP 的试样不能随机分到训练集和测试集。
2. Mean、Ridge、Random Forest、SVR 分别在验证什么假设。
3. MAE、RMSE、R2 和预测区间应如何共同解释。
4. 为什么论文分数与本项目分数不能直接比较。
5. 哪些结果可以进入后续代理模型，哪些结果说明需要补数据或补特征。

## 2. 问题定义

输入向量固定为四个源数据字段：

\[
x = [V_{CNT},\;V_{interlayer},\;T_g,\;P_{mfg}]
\]

分别预测：

- 弯曲强度；
- 源文件标注的弯曲模量；
- 拉伸强度；
- 源文件标注的 II 型能量释放率。

`material_type_id` 只用于分组，不作为输入特征。否则模型可能直接记住材料类别，而不是学习工艺和结构参数到性能的关系。

这仍然是宏观性能表格回归，不是应力-应变本构训练，也不能直接生成 Abaqus 材料卡。

## 3. 外层验证为什么按材料类型留一

每一折完整留出一个 `material_type_id`：

```text
外层训练：其余材料类型的全部有效样本
外层测试：当前被留出的完整材料类型
```

所有汇总指标都来自外层 OOF（out-of-fold）预测。一个有效样本只测试一次，且其材料类型从未出现在对应训练集。

这回答的是更接近工程应用的问题：模型遇到一种没训练过的 CFRP 体系时还能否泛化。逐行随机切分回答的只是“同类试样之间能否插值”，通常会更乐观。

## 4. 四个基线分别检查什么

| 模型 | 作用 | 本项目固定设置 |
|---|---|---|
| Mean | 不使用输入，只预测外层训练集均值 | 判断复杂模型是否真的有价值 |
| Ridge | 标准化后的带 L2 正则线性回归 | `alpha=1.0` |
| Random Forest | 学习非线性和变量交互 | 250 棵树，`min_samples_leaf=2` |
| SVR | 小样本非线性核回归 | 标准化输入和目标，RBF 核，`C=10` |

本轮不做超参数搜索。若使用测试折选择参数，会把测试信息泄漏回训练流程。后续调参必须放在每个外层训练集内部的嵌套验证中。

## 5. 预测区间如何生成

外层训练集内部再次按材料类型留一，得到内层 OOF 绝对残差：

```text
外层测试材料 A
  外层训练材料 B...I
    内层依次留出 B...I，收集只由其余材料预测得到的绝对残差
  用这些残差的有限样本分位数构造 A 的对称预测区间
```

因此 A 的真实值既不参与模型训练，也不参与自己的区间宽度校准。默认名义覆盖率为 90%。这种区间用于暴露小样本和跨材料偏移风险，
不是认证置信区间或设计允许值；实际覆盖率低于 90% 时必须明确报告欠覆盖。

## 6. 运行完整实验

先准备官方数据：

```powershell
conda run -n pylabfea materialai-cfrp-data --accept-license
```

再运行四目标、四模型基线：

```powershell
conda run -n pylabfea materialai-cfrp-baseline
```

只运行一个教学目标：

```powershell
conda run -n pylabfea materialai-cfrp-baseline `
  --target flexural_strength_mpa `
  --model mean `
  --model ridge
```

## 7. 输出目录

```text
workspace/experiments/cfrp_grouped_baselines/<run_id>/
├─ predictions.csv
├─ fold_metrics.csv
├─ model_comparison.csv
├─ summary.json
├─ run_manifest.json
├─ REPORT_CN.md
├─ figures/
├─ models/
└─ model_cards/
```

- `predictions.csv` 是最重要的证据表，保存每个样本的 OOF 预测、残差、区间和测试材料类型。
- `fold_metrics.csv` 用于定位哪一种 CFRP 材料类型最难预测。
- `model_comparison.csv` 才是汇总表，不能脱离前两张表单独使用。
- `models/` 是用全部有效样本重训后的推理模型；它们不参与汇总指标计算。
- `run_manifest.json` 保存输入哈希、固定参数、运行库版本和全部产物哈希。

## 8. 默认配置的实测基线

以下结果来自官方数据版本 1、`random_state=42`、250 棵随机森林和 90% 名义区间。指标均为按材料类型留一的逐样本 OOF 结果。

| 目标 | 分组 MAE 最优 | MAE | RMSE | R2 | 相对 Mean 的 MAE 改善 | 区间覆盖率 |
|---|---|---:|---:|---:|---:|---:|
| 弯曲强度 | Ridge | 40.676 MPa | 54.803 MPa | 0.839 | 70.2% | 77.4% |
| 源标签弯曲模量 | Ridge | 4.452 | 5.478 | 0.697 | 51.1% | 87.1% |
| 拉伸强度 | Random Forest | 6.000 MPa | 7.554 MPa | -0.077 | 10.4% | 95.2% |
| 源标签 II 型能量释放率 | Mean | 419.159 | 560.684 | -0.603 | 0% | 85.7% |

### 结果解释

- 四个输入对弯曲强度和源标签弯曲模量具有明显跨材料预测信息，Ridge 反而优于更复杂的模型。
- 拉伸强度的 Random Forest 虽把 MAE 降低约 10%，但 R2 仍为负，说明相对目标总体波动，跨材料排序能力不足。
- II 型能量释放率没有任何 ML 模型超过 Mean 的 MAE，现有特征不足以描述其跨材料差异。
- 弯曲强度最优模型的实际区间覆盖率只有 77.4%，低于名义 90%，说明点预测尚可并不等于不确定性已经可靠。

这不是“项目失败”。它准确指出了下一轮数据需求：层间材料体系、界面参数、铺层和断裂试验条件必须进入 II 型断裂能模型。

## 9. 与论文报告值的关系

原论文报告弯曲强度、弯曲模量和 II 型能量释放率的最佳 R2 分别约为 0.966、0.871 和 0.903。论文使用逐行留一法；本项目完整留出材料类型。
两者测试问题不同，因此不能用数值大小直接声称复现成功或失败。

本项目当前状态登记为 `baseline_completed`：表示官方数据和模型链条已经独立运行并留下可审计指标，但没有冒充逐项复现论文训练协议。

## 10. 面试时如何讲

可以用下面的逻辑，而不是只背一个 R2：

> 我先复核公开 CFRP 数据的许可、哈希、缺失值和单位，再按材料体系生成固定留一折，避免同类试样泄漏。随后比较 Mean、Ridge、随机森林和 SVR，
> 并在每个外层训练集内部再次按材料体系校准预测区间。结果显示简单 Ridge 对弯曲强度的跨材料 R2 仍有 0.839，而 II 型断裂能无法超过均值基线。
> 这说明前者已有可用结构信号，后者必须补充界面和铺层特征。下一步不是盲目堆神经网络，而是用 RVE/Abaqus 和真实试验补齐物理信息。

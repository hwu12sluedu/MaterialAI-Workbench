# 复合材料代理模型：从会用到学透

本文面向具有机械、材料力学和 Abaqus 经验，但刚开始系统学习 Python 与机器学习的工程师。目标不是只训练出一条好看的拟合曲线，而是建立下面这个可验证闭环：

```text
微观结构参数
-> Abaqus 三相 RVE 与六工况 PBC
-> 等效材料参数
-> Abaqus 三维结构模型
-> ODB 真实响应
-> 受治理的数据集
-> 机器学习代理模型
-> 未见工况 Abaqus 验证
-> 客户端预测与适用范围提示
```

## 1. 最终要预测什么

“复合材料代理模型”不是一个单独模型。产品按难度分为三层。

### 1.1 微观到材料参数

输入示例：

- 纤维体积分数 `Vf`
- 纤维、基体和界面的材料参数
- 纤维方向、半径、长度和随机种子
- RVE 尺寸与网格信息

输出示例：

- `E1/E2/E3`
- `G12/G13/G23`
- `nu12/nu13/nu23`
- 后续扩展的强度、损伤起始和演化参数

这是第一优先级。标签应主要来自真实 RVE/PBC 求解，而不是混合定则估算。

### 1.2 材料与结构参数到工程响应

输入示例：

- 上一层得到的正交各向异性材料参数
- 板长、板宽、厚度、孔径
- 载荷、应变、铺层方向和网格尺度

输出示例：

- 孔边最大应力
- 最大位移与反力
- Tsai-Wu、Hashin 等失效指标
- 初始损伤载荷或极限载荷

这一层用于快速筛选设计，再把候选方案送回 Abaqus 复核。

### 1.3 参数到完整场

未来输入还可以包含几何或体素结构，输出节点/单元上的应力场、应变场、位移场。可研究 CNN、GNN、DeepONet 或神经算子，但它不应作为第一个模型。先把标量代理模型的数据与验证做扎实。

## 2. 当前产品已经做到哪里

| 能力 | 当前状态 | 应如何理解 |
|---|---|---|
| Fiber/Interface/Matrix 三相 RVE | 已实现 | 可生成微观几何、phase map 和 Abaqus 输入 |
| 六工况 PBC | 已实现可选流程 | 可提交 RVE Job 并从 ODB 计算等效刚度 |
| 等效材料映射 | 已实现 | 可生成正交各向异性材料卡并写入宏观模型 |
| 三维带孔板 | 已实现 | 可生成、求解并提取 ODB 响应 |
| 批量 DOE | 已实现 | 可改变 `Vf`、界面效率、孔径和微观随机种子 |
| 代理模型 | 已实现 | 支持 Random Forest、MLP、留出评估和 RF 不确定性 |
| 客户端入口 | 已实现 | 可查看 RVE、批次、数据集和代理模型 |
| 工程预测精度 | 尚未证明 | 真实 PBC/ODB 标签数量不足，当前部分标签仍是解析估算 |
| 完整场预测 | 尚未实现 | 属于完成标量闭环后的研究方向 |

结论：当前版本能够证明流程、接口和产品形态，但不能把已有演示指标写成工业精度结论。

## 3. 一条样本是怎样产生的

以“预测孔边最大应力”为例：

1. 固定单位制，例如 `mm-N-s-MPa`。
2. 采样 `Vf`、组分参数、纤维方向和随机种子。
3. 生成三相 RVE，而不是只计算等效混合定则。
4. 对 RVE 执行 3 个拉伸和 3 个剪切 PBC 工况。
5. 从 ODB 体积平均应力与宏观应变得到等效刚度。
6. 将等效参数写入三维带孔板正交各向异性材料卡。
7. 求解宏观模型并提取孔边应力、位移、反力和失效指标。
8. 通过 Job、ODB、单位、收敛和物理范围质量门。
9. 只有质量门合格的结果才能进入训练集。

这也是判断“有限元数据是否能训练机器学习”的核心：不是有 CSV 就能训练，而是每一行必须能追溯到输入、求解器证据和结果提取规则。

## 4. 你应该怎样学习

每天建议投入 60 到 90 分钟：

| 环节 | 时间 | 产出 |
|---|---:|---|
| 理论 | 15 分钟 | 用自己的话解释一个概念 |
| 读代码 | 20 分钟 | 找到输入、输出和调用关系 |
| 动手 | 25 分钟 | 跑一个命令或改一个参数 |
| 验证 | 15 分钟 | 看测试、图、CSV 或 ODB 证据 |
| 复盘 | 5 分钟 | 写下今天理解和未理解的问题 |

不要按文件从头背代码。每次围绕一个工程问题逆向阅读：

```text
界面上的一个按钮
-> 调用了哪个 workflow
-> workflow 产生了什么文件
-> 文件中的一个数来自哪个公式或 ODB 字段
-> 哪个测试保证它没有被改坏
```

## 5. 八周学习与开发闭环

### 第 1 周：看懂现有链路

- 运行客户端并定位“复合材料 RVE”和“代理模型”。
- 阅读 `CompositePlateConfig`。
- 区分组分参数、微观几何参数、宏观结构参数。
- 看懂一行 `composite_plate_dataset_row.csv`。
- 验收：能画出当前数据流，并指出哪些标签是估算、哪些来自 ODB。

### 第 2 周：微观 RVE

- 理解 RVE、代表性、随机种子和体积分数。
- 看懂 Fiber/Interface/Matrix 三相分区。
- 改变 `Vf` 与纤维方向，观察 phase map 和网格。
- 验收：能解释为什么不同随机微结构不能随意分到训练集和测试集两边。

### 第 3 周：PBC 与均匀化

- 理解周期性边界和六个独立应变工况。
- 读懂平均应力除以宏观应变得到刚度的步骤。
- 跑 1 个真实 RVE 的六工况闭环。
- 验收：能从 ODB 证据手算复核至少一个等效模量。

### 第 4 周：数据集治理

- 设计参数范围和 DOE。
- 加入单位、版本、网格、随机种子、收敛状态和文件指纹。
- 建立训练资格门禁。
- 验收：失败 Job、缺 ODB 或估算标签不能冒充真实训练数据。

### 第 5 周：代理模型基础

- 用 Random Forest 建立基线。
- 理解训练集、验证集、测试集和数据泄漏。
- 学习 MAE、RMSE、R2、相对误差和残差图。
- 再训练 MLP，并理解为什么需要标准化。
- 验收：能解释某个模型为什么优于另一个模型，而不是只报一个 R2。

### 第 6 周：工程验证与不确定性

- 预留从未参加训练的微观随机种子和参数区间。
- 对代理模型推荐点重新运行 Abaqus。
- 标记插值、边界和外推请求。
- 输出预测区间或模型分歧。
- 验收：产品能在“不知道”时拒绝给出过度自信结果。

### 第 7 周：客户端产品化

- 在 App 中选择数据集、特征、标签和模型。
- 输入一组新参数并显示预测、区间和最近训练样本。
- 一键生成 Abaqus 复核任务，但提交前必须人工确认。
- 验收：普通用户不读源码也能完成一次预测与复核。

### 第 8 周：讲清楚并发布

- 从输入、物理、数据、模型、验证五个层面讲项目。
- 完成一个带孔板闭环案例报告。
- 复跑公开测试清单。
- 更新 GitHub、演示视频、中文教学和简历表述。
- 验收：你能够独立演示，并回答“模型在哪里有效、在哪里无效”。

## 6. 第一次动手

### 6.1 只创建批量计划，不启动 Abaqus

```powershell
cd D:\githubproject\pyLabFEA
conda run -n pylabfea python -m material_ai_workbench.run_composite_batch create --name learn_composite --sample-count 12
```

查看终端返回的计划目录，打开：

- `composite_batch_plan.json`
- `samples.csv`

回答三个问题：

1. 哪些列是输入特征？
2. 哪些参数目前没有变化，因此模型无法学习它们的影响？
3. `random_seed` 为什么是物理数据的一部分，而不只是编程细节？

### 6.2 运行一个不提交 Abaqus 的流程样本

```powershell
conda run -n pylabfea python -m material_ai_workbench.run_composite_batch run <计划目录> --max-samples 1
```

此步骤主要理解文件流。它生成的估算标签只能用于学习流程，不能作为最终工程真值。

### 6.3 训练基线模型

```powershell
conda run -n pylabfea python -m material_ai_workbench.run_composite_batch train <计划目录>\composite_dataset.csv --model-kind random_forest --uncertainty ensemble
```

只有在明确知道标签来源后才解释指标。样本很少时，结果只能证明程序可运行。

### 6.4 真实 PBC 与宏观 Abaqus

真实求解需要 Abaqus 许可证、Socket Bridge 或本机命令环境正常。先用 1 个样本验收，再扩展批次：

```powershell
conda run -n pylabfea python -m material_ai_workbench.run_composite_batch create --name real_pbc_gate --sample-count 1 --run-pbc-homogenization --use-abaqus-homogenization --run-abaqus
```

创建计划不会立即把 dry-run 当作真实数据。执行前应检查计划目录、单位、网格和 Job 数量。

## 7. 工程精度门槛

进入稳定版前至少满足：

- 训练标签来源能区分 `estimate`、`pbc_odb` 和 `macro_odb`。
- 测试样本按微结构随机种子或设计族分组，避免数据泄漏。
- 至少包含训练域内部、边界和域外三类验证点。
- 同时报告 MAE、RMSE、相对误差、最大误差和残差。
- 预测结果满足基本物理范围与材料对称性约束。
- 代理模型输出训练域距离或外推告警。
- 任何关键预测都能生成对应 Abaqus 复核任务。

样本数量没有通用魔法值。建议先用 12 个样本调试流程、60 到 100 个真实样本建立第一版可信基线，再根据学习曲线决定是否扩展到数百个样本。最终标准是独立 Abaqus 误差收敛，而不是达到某个固定数量。

## 8. 每日自动任务与人工学习如何配合

每日自动任务只完成一个小增量，并输出：

1. 今天解决的一个问题。
2. 修改的代码与测试。
3. 一段面向机械工程师的中文讲解。
4. 一个你需要亲手完成的小练习。
5. 当前剩余风险和明天建议。

自动任务不能替代你的学习。你每天只需要完成两件事：读当天讲解，并亲手做当天练习。每周末再由我们一起复盘一周的提交、测试证据和概念掌握情况。

## 9. 真正学会的验收标准

当你可以不看提示完成以下工作，才算学透：

- 从一个工程问题定义特征、标签、参数范围和单位。
- 判断 RVE 是否具有代表性并解释 PBC。
- 从 Abaqus ODB 验证一个等效参数或结构响应。
- 发现数据泄漏、错误标签和外推风险。
- 独立训练 RF 与 MLP，并解释指标差异。
- 把代理模型预测送回 Abaqus 做未见工况验证。
- 向同事说明产品能做什么、不能做什么以及为什么。

# 功能边界与验收说明

本页区分“代码已经实现”“需要本机软件验证”和“尚未承诺”的能力，避免把研究流程写成工业认证结果。

## 1. 材料模型

| 能力 | 当前状态 | 验收方式 |
|---|---|---|
| J2 屈服训练 | 已实现 | 固定随机种子训练，检查分类指标、屈服面和导出文件 |
| Hill 各向异性 | 已实现 | 校验六个方向比值、异常输入和训练回归测试 |
| Barlat 实验 | 已实现研究级路径 | 对照参考工况和 F1 指标；不能替代完整 Yld2000/Yld2004 标定 |
| Neo-Hookean / Mooney-Rivlin | 已实现曲线与材料卡路径 | 检查参数、应力曲线和导出格式；不使用 SVC 训练超弹性 |
| 用户实验曲线 | 已实现导入校验 | 检查单调性、重复点、NaN、单位和弹性段拟合 |

材料模型输出不是材料认证数据。进入设计计算前必须使用真实试验、文献来源、单位约定和独立算例复核。

## 2. Abaqus 工作流

| 能力 | 无 Abaqus | 有 Abaqus 与许可证 |
|---|---|---|
| INP/建模脚本生成 | 可用 | 可用 |
| 三维带孔板验收 | 可生成 prepared 工件 | 可生成 CAE/INP、提交 Job、提取 ODB 并归档 |
| Job 计划和队列 | 可用 | 可提交并监控 |
| ODB/CSV 后处理 | 只能读取已有兼容数据 | 可通过 SMAPython/MCP 提取 |
| CAE 实时查询 | 不可用 | 启动 MCP Socket Bridge 后可用 |
| 结果报告 | 生成输入与流程证据 | 可加入真实求解结果和云图 |

客户端不会把“脚本已生成”显示成“求解已完成”。状态必须来自 Job/ODB 证据。

三维验收状态固定为 `prepared -> built -> solved -> postprocessed -> validated -> archived`。阶段失败可从同一运行目录恢复；恢复前后的配置、日志和文件路径都写入 `acceptance_manifest.json`。

## 3. 复合材料微观模型

当前 RVE 表示 Fiber、Interface、Matrix 三相微观结构，并保存纤维取向、体积分数、相图和六工况输入。它不是简单的宏观等效材料卡。

仍需针对具体研究继续验证：

- 周期性几何与严格 PBC 配对；
- 圆柱纤维的网格质量和周期截断；
- 界面本构、损伤与失效参数；
- RVE 尺寸、随机种子和统计代表性；
- 微观均匀化结果到宏观三维模型的映射误差。

## 4. 代理模型与自然语言

- Random Forest、MLP、时间序列和多保真模块已经具备训练、评估与保存流程。
- 数据集必须区分训练、验证和独立测试，报告样本数量、特征来源和误差分布。
- 小样本高分不代表可泛化到新结构、新材料或新载荷。
- 自然语言模块只生成受 schema 约束的任务；任意复杂 CAD、接触、工艺 Step 和用户子程序仍需工程师建模。
- 外部模型服务不可直接获得不受限制的命令执行权限。

## 5. 发布验收

每个公开版本至少满足：

1. 快速测试集通过。
2. 源码桌面启动健康检查通过。
3. Windows 便携包在干净构建机完成冻结后健康检查。
4. Release 同时提供 ZIP、SHA256、wheel 和源码包。
5. 发布候选不含 `.env`、ODB/CAE、用户案例、明文密钥或内部任务记录。
6. 带 Abaqus 的发布候选至少完成一个三维带孔板实机闭环，并保存 `.sta`、ODB 特征和验收清单。

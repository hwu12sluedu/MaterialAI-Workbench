# 案例库中文使用指南

## 目标

案例库把每天完成的 Abaqus 仿真沉淀为可检索、可复用、可训练的数据资产。它不复制大体积 ODB/CAE 文件，而是索引路径、提取 INP/CSV/日志特征，并保存工程备注。

## 推荐工作流

1. 完成一个 Abaqus 仿真案例。
2. 保留该案例文件夹中的 `.inp`、`.odb`、`.sta`、`.msg`、`.csv`、图片和报告。
3. 在 Streamlit App 的 `案例库` 页面输入案例文件夹或单个 `.inp` 路径。
4. 填写标题、标签、状态、经验总结和下一步动作。
5. 点击扫描归档。
6. 如有 ODB，使用 MCP 或 SMAPython 提取字段统计和逐帧曲线。
7. 导出案例库训练数据集。
8. 在 `代理模型` 页面训练 RandomForest 或 MLP 代理模型。

## 案例库会保存什么

- `case_summary.json`：结构化元数据。
- `case_report.md`：人可读报告。
- 文件清单：模型、结果、数据、报告、脚本分类。
- INP 特征：节点/单元估算、材料、step、载荷、边界、输出请求。
- 结果特征：CSV 数值列统计、日志 warning/error、ODB 路径索引。
- ODB 后处理结果：字段统计、逐帧曲线、报告路径。
- 工程参数：屈服强度、弹性模量、泊松比、批量计划 ID 等。

## 相似案例检索

`find_similar_cases()` 会基于可解释的数值特征计算相似度，包括文件数、INP 节点/单元数量、CSV 行数、最大 Mises/PEEQ 和材料参数。未来自然语言仿真会先检索相似案例，再生成新 job 和参数建议。

## 数据质量建议

- 每个成功案例至少保留 `.inp`、`.sta`、关键云图、后处理 CSV 和简短说明。
- 对失败案例也要归档，标签写 `failed` 或 `needs_review`，因为失败样本能帮助 AI 识别风险。
- 每次后处理尽量提取同一套字段，例如 `S`、`PEEQ`、`U`、`RF`，便于训练代理模型。
- 批量仿真时记录参数范围，避免训练数据只有结果没有输入条件。

# MaterialAI Workbench v0.1.0 发布状态

## 发布定位

`v0.1.0` 是首个可公开安装、可运行、可追溯的工程 MVP。它不是“全自动替代工程师”的成品，而是把材料机器学习、复合材料微观建模、Abaqus 任务准备、结果提取和案例数据沉淀接入同一工作区的可靠起点。

## 已交付闭环

1. 从目标体积分数、纤维角度、角度离散度、长度和直径生成可复现的三维 Fiber/Interface/Matrix 布局。
2. 同一份 `fiber_layout.json` 同步驱动三维显示、二维投影、体素 phase map、Abaqus 微观模型和机器学习特征，避免“图是一套、计算又是一套”。
3. 自动生成六个均匀化载荷工况、三维带孔板 Abaqus 建模脚本、ODB 后处理脚本、数据行和工程报告。
4. 支持实验曲线与 Abaqus CSV 导入、案例索引、Job 队列、MCP 状态查询和 ODB/截图提取。
5. 支持 J2、Hill、Barlat 机器学习屈服实验，以及 Random Forest、MLP、时序和多保真代理模型基线。
6. 支持将自然语言转换成可校验任务计划；用户确认后才允许进入受支持的执行路径。
7. 提供 Streamlit 客户端、Python API、命令行入口、Docker 配置和完整中文文档入口。

## 必须诚实区分的状态

- `generated`：已经生成 Abaqus 脚本或 INP，但本次没有提交求解。
- `submitted/running`：任务已进入本机 Abaqus，尚不能据此宣称验证通过。
- `completed`：求解正常结束并获得可读取结果。
- `validated`：结果通过网格、边界条件、单位、收敛、守恒或参考数据等工程检查。

因此，公开版本不会把脚本生成、代理模型拟合分数或示例云图包装成真实工业验证结论。

## 当前边界

1. 微观模型目前采用体素化三相表示；严格周期 wrap、真实圆柱网格质量和 PBC 基准仍需专项验证。
2. 带孔板是统一的最小宏观验证载体，不代表已经覆盖所有材料、接触、损伤和非线性工况。
3. LLM 负责计划与参数建议，不直接执行任意 Python 或不受控 Abaqus 命令。
4. 代理模型只有在真实案例达到足够规模并完成独立测试集验证后，才可用于工程预测。
5. Abaqus、MCP、商业材料数据与大型 ODB 都是本地可选资源，不随 GitHub 仓库分发。

## 下一版本主线

`v0.2.0` 聚焦真实 Abaqus 证据链：严格 PBC 基准、RVE 求解结果回写、带孔板材料映射、ODB 特征提取、案例质量评级，以及使用真实仿真数据训练并验证代理模型。桌面封装将在核心计算与数据契约稳定后推进。
